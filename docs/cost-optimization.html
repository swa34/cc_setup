Cost Optimization

Claude Code consumes tokens for every interaction. The average developer spends ~$6/day, with 90% of users staying under $12/day. This chapter shows you how to keep costs predictable — from session hygiene to team budgeting at scale.

Token Budgeting Intermediate

Key Rule

A 200K-token conversation resends ALL context with every prompt. Each new message costs proportionally more as context grows. The context window is the most important resource to manage.

Token costs scale with context size: the more context Claude processes, the more tokens you use. Claude Code automatically optimizes costs through prompt caching (which provides up to a 90% discount for repeated content like system prompts) and auto-compaction (which summarizes conversation history when approaching context limits).

Session Hygiene

  • /clear between tasks — Start fresh when switching to unrelated work. Stale context wastes tokens on every subsequent message. Use /rename before clearing so you can /resume back to it later.
  • Focused prompts naming exact files — "Fix the TypeScript errors in src/api/auth.ts" is far cheaper than "fix all the errors." Vague prompts trigger broad file scanning.
  • Batch related changes — Instead of 5 separate prompts for 5 related fixes, describe all 5 in one prompt. Each prompt resends the full context.
  • /compact with instructions — When context gets large, run /compact Focus on the API changes and test output to tell Claude what to preserve during summarization.
  • Use /btw for side questions — Quick questions answered in an overlay that does not consume context tokens in your main conversation.

Track your spending with /cost to see token usage for the current session, or configure your status line to display context usage continuously.

Model Selection Guide Intermediate Docs

Model choice is the single biggest lever for controlling costs. Sonnet handles ~80% of coding tasks well at a fraction of Opus's price. Reserve Opus for complex architectural decisions or multi-step reasoning where Sonnet falls short.

ModelInput $/MTokOutput $/MTokBest For
Sonnet 4.6 $3 $15 General coding, ~80% of tasks. Fast, capable, cost-effective.
Opus 4.6 $15 $75 Complex architecture, multi-file reasoning, difficult debugging.
Haiku $0.80 $4 Simple queries, subagents, background tasks.
Prompt Caching Discount

Cached input tokens are billed at a 90% discount. Since system prompts, CLAUDE.md, and conversation history are heavily cached, your effective input cost is often much lower than the listed rate.

Choosing the Right Model

Switch models mid-session with /model, or set a default in /config. The opusplan alias provides an automated hybrid: Opus for plan mode reasoning, Sonnet for code execution.

Extended Thinking Advanced

Extended thinking is enabled by default because it significantly improves performance on complex planning and reasoning tasks. However, thinking tokens are billed as output tokens — the most expensive token type. The default budget can be tens of thousands of tokens per request.

Controlling Thinking Costs

MethodHowEffect
Effort level /effort low, /effort medium, /effort high Dynamically allocates thinking based on task complexity. Persists across sessions.
Max effort /effort max Deepest reasoning, no constraint on thinking tokens. Opus 4.6 only. Current session only.
Budget cap MAX_THINKING_TOKENS=10000 Hard limit on thinking tokens per request. Only applies when adaptive reasoning is disabled.
Disable thinking Toggle in /config Turns off extended thinking entirely. Faster but less capable.
Disable adaptive CLAUDE_CODE_DISABLE_ADAPTIVE_THINKING=1 Reverts to fixed thinking budget controlled by MAX_THINKING_TOKENS.

When to Adjust

  • Use high/max effort for complex architectural decisions, multi-file refactors, and difficult debugging sessions where deep reasoning pays off.
  • Use low effort for routine tasks: renaming variables, adding log lines, simple queries. Thinking tokens have diminishing returns on simple work.
  • Use medium effort (the default for Opus on subscription plans) as a balanced starting point for most tasks.
Quick Check

The current effort level is displayed next to the logo and spinner (e.g., "with low effort") so you can confirm which setting is active without opening /model.

Reducing MCP Overhead Advanced

Each MCP server adds its tool definitions to your context window — even when those tools are sitting idle. If you have 5 MCP servers configured, you might be paying for thousands of tokens of tool descriptions in every single message, even if you never use them.

Diagnosis

Run /context to see what is consuming space in your context window. If MCP tool definitions are a significant fraction, you have optimization opportunities.

Strategies

  • Prefer CLI tools over MCP — Tools like gh, aws, gcloud, and sentry-cli are more context-efficient because they do not add persistent tool definitions. Claude can run CLI commands directly via Bash without the overhead. If a CLI tool exists for the service, it is almost always cheaper than the equivalent MCP server.
  • Disable unused servers — Run /mcp to see configured servers and disable any you are not actively using in the current session.
  • Enable tool search with a lower threshold — When MCP tool descriptions exceed a percentage of your context window, Claude Code automatically defers them and loads tools on-demand. Set a lower threshold to be more aggressive: 
    # Defer tools when they exceed 5% of context (default is 10%)
export ENABLE_TOOL_SEARCH=auto:5
    Deferred tools only enter context when actually used, so idle tool definitions stop consuming space.
Example

A GitHub MCP server might add 2,000+ tokens of tool definitions. Using the gh CLI instead adds zero persistent overhead — Claude just runs gh pr list, gh issue view 123, etc. as regular Bash commands.

CLAUDE.md as Index Intermediate

Your CLAUDE.md file is loaded into context at the start of every session. Because of prompt caching, this content gets a 90% discount on input tokens — making it one of the most cost-effective ways to give Claude context. But there is a tradeoff: every line in CLAUDE.md is present in every message, so bloat still costs.

The Index Pattern

Keep CLAUDE.md under ~500 lines by treating it as an index that points to detailed reference files. Claude reads the index every session but only loads referenced files when relevant:

# Project: my-api

## Build & Test
- `npm run build` — compile TypeScript
- `npm test` — run Jest suite (use --testPathPattern for single files)
- `npm run lint` — ESLint + Prettier check

## Code Style
- ES modules (import/export), not CommonJS
- Destructure imports: `import { foo } from 'bar'`
- Functional components with hooks, no class components

## Architecture
- See @docs/architecture.md for system overview
- See @docs/api-conventions.md for REST patterns
- See @docs/database.md for migration procedures

## Skills (loaded on demand)
- /fix-issue — Analyze and fix a GitHub issue
- /review — Code review with security focus
- /migrate — Database migration workflow

This pattern gives Claude the essentials (~100-200 tokens) while domain knowledge lives in skills and referenced files that load on-demand.

Pruning Test

For each line in your CLAUDE.md, ask: "Would removing this cause Claude to make mistakes?" If not, cut it. Move specialized instructions into skills that load on-demand only when invoked.

Team Budgeting Advanced Docs

When using the Claude API, you can set workspace spend limits on the total Claude Code workspace spend through the Anthropic Console. Admins can view cost and usage reporting to track team-wide consumption.

Rate Limit Recommendations by Team Size

The TPM (Tokens Per Minute) per user decreases as team size grows because fewer users tend to use Claude Code concurrently in larger organizations. These limits apply at the organization level, not per individual user.

Team SizeTPM per UserRPM per User
1-5 users200k-300k5-7
5-20 users100k-150k2.5-3.5
20-50 users50k-75k1.25-1.75
50-100 users25k-35k0.62-0.87
100-500 users15k-20k0.37-0.47
500+ users10k-15k0.25-0.35

For example, 200 users at 20k TPM per user = 4 million total TPM (200 * 20,000). Individual users can temporarily consume more than their calculated share when others are not actively using the service.

Rollout Strategy

  1. Start with a small pilot — 5-10 developers for 2-4 weeks to establish usage patterns.
  2. Monitor with OpenTelemetry — Track claude_code.cost.usage and claude_code.token.usage per user and team.
  3. Set conservative limits — Start at the lower end of the recommended range and adjust upward based on actual usage.
  4. Educate on cost hygiene — Share the techniques from this page with your team. The biggest wins come from /clear between tasks and choosing the right model.
  5. Expand gradually — Roll out to additional teams once you have a baseline cost model.
Monthly Budget Estimate

On average, Claude Code costs ~$100-200 per developer per month with Sonnet 4.6, though there is large variance depending on how many instances users are running and whether they use it in automation. Agent teams use approximately 7x more tokens than standard sessions.

Quick Reference Intermediate

A cheat sheet of the highest-impact cost optimization techniques, sorted by effort required.

#TechniqueImpactEffort
1 /clear between tasks High — eliminates stale context from every subsequent message None
2 Use Sonnet by default High — 5x cheaper than Opus for input, 5x for output None
3 Write specific prompts High — fewer file reads, less scanning, less wasted context Low
4 Lower effort for simple tasks Medium — saves thousands of thinking tokens per request None
5 Prefer CLI tools over MCP Medium — removes persistent tool definitions from context Low
6 Keep CLAUDE.md concise Medium — reduces base context loaded in every message Low
7 Use subagents for investigation Medium — verbose output stays in subagent context, not yours Low
8 Disable unused MCP servers Medium — each idle server wastes tokens on tool definitions None
9 Use /compact with focus instructions Medium — preserves what matters, discards what does not Low
10 Move specialized instructions to skills Low-Medium — reduces base CLAUDE.md size, loads on demand Medium
The 80/20 Rule

Techniques 1-3 alone — clearing between tasks, using Sonnet, and writing specific prompts — account for the majority of cost savings. Master these first before optimizing MCP servers or thinking budgets.

Background Usage

Claude Code uses a small number of tokens for background functionality (conversation summarization, command processing) even without active interaction — typically under $0.04 per session. This is normal and not a cost concern.
