Getting Started with Claude Code

Everything you need to go from zero to productive with Claude Code — Anthropic's agentic coding assistant that lives in your terminal.

What is Claude Code Beginner

Claude Code is a command-line tool that brings Claude directly into your terminal. Unlike chat-based AI assistants, Claude Code is agentic — it can read files, write code, run commands, search the web, and iterate on its own work.

Think of it as a senior engineer pair-programming with you, except it can also grep your entire codebase, run your tests, and fix what it broke — all in one conversation.

Key Concept

Claude Code uses an agentic loop: it receives your prompt, decides which tools to use, takes action, observes the result, and repeats until the task is done. You approve each step (or configure it to run autonomously).

Claude Code also integrates with VS Code, JetBrains IDEs, and has a Desktop app. The CLI is the foundation — IDE integrations wrap it with additional UI.

First Run & Authentication Beginner

Navigate to any project directory and run claude:

Terminal
$ cd ~/my-project $ claude Welcome to Claude Code v1.x Authenticate at: https://claude.ai/login?... ✓ Authenticated successfully $ claude /init Analyzing your project... ✓ Generated CLAUDE.md with project context

On first run, Claude Code opens a browser window for authentication. After that, run /init to auto-generate a CLAUDE.md file — this gives Claude context about your project's tech stack, build commands, and conventions.

Understanding the Agentic Loop Beginner Docs

Unlike traditional chat AI, Claude Code has tools it can use autonomously. When you give it a task, it:

  1. Gathers context — reads files, searches code, checks git history
  2. Takes action — edits files, runs commands, creates tests
  3. Verifies results — runs tests, checks for errors, reviews its own output
  4. Iterates — if something fails, it tries again with a different approach

You control how much autonomy Claude has through permission modes:

Default Mode
Claude asks permission before each write or command. Safest for learning and sensitive repos.
Accept Edits
Auto-approves file edits but still asks before running shell commands.
Don't Ask
Full autonomy within allowed tools. Best for trusted, sandboxed environments.
