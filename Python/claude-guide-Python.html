UV — The Modern Python Tool Docs

UV is a fast, all-in-one Python package and project manager from Astral (the Ruff team). It replaces pip, pip-tools, pyenv, virtualenv, and pipx — all in a single binary written in Rust.

Install UV

# Install UV
curl -LsSf https://astral.sh/uv/install.sh | sh

# Or via Homebrew
brew install uv

# Verify
uv --version
# Install UV (PowerShell)
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

# Or via WinGet
winget install --id=astral-sh.uv

# Verify
uv --version

Why UV over pip/poetry/pipenv?

Feature UV pip + venv Poetry
Speed 10-100x faster Baseline Slow resolver
Lockfile uv.lock (cross-platform) None (pip-tools needed) poetry.lock
Python management Built-in (uv python install) Needs pyenv Needs pyenv
Tool runner uvx (like npx) Needs pipx No
Single binary Yes (Rust) No No

Project Setup with UV Docs

Create a new project

# Create a new project (creates pyproject.toml, src/, etc.)
uv init my-project
cd my-project

# Or initialize in an existing directory
cd existing-dir
uv init

What UV creates

my-project/
  pyproject.toml    # Project metadata + dependencies
  .python-version   # Pinned Python version
  src/
    my_project/
      __init__.py
  README.md

pyproject.toml — the single config file

UV uses pyproject.toml as the single source of truth — no setup.py, setup.cfg, requirements.txt, or Pipfile. 

[project]
name = "my-project"
version = "0.1.0"
description = "My Python project"
readme = "README.md"
requires-python = ">=3.12"
dependencies = [
    "fastapi>=0.115",
    "httpx>=0.28",
    "pydantic>=2.10",
]

[dependency-groups]
dev = [
    "pytest>=8.3",
    "ruff>=0.9",
    "mypy>=1.14",
]

Managing Python versions

# List available Python versions
uv python list

# Install a specific version
uv python install 3.12

# Pin version for this project
uv python pin 3.12

Virtual environments

UV creates and manages .venv automatically. You rarely need to think about it.

# UV auto-creates .venv when you run commands
uv run python script.py   # Creates .venv if needed, runs script

# Or explicitly create one
uv venv                   # Creates .venv in current dir
uv venv --python 3.12     # With a specific Python version

# Activate manually (if needed for IDE integration)
source .venv/bin/activate        # Linux/macOS
.venv\Scripts\activate           # Windows

Managing Dependencies Docs

Add dependencies

# Add a production dependency
uv add fastapi
uv add "httpx>=0.28"

# Add a dev dependency
uv add --group dev pytest ruff mypy

# Add multiple at once
uv add sqlalchemy alembic psycopg2-binary

Remove dependencies

uv remove httpx
uv remove --group dev mypy

The lockfile: uv.lock

UV automatically generates and maintains uv.lock — a cross-platform lockfile that pins exact versions for reproducible builds. Commit it to git.

# Sync your environment to match the lockfile
uv sync

# Sync including dev dependencies
uv sync --all-groups

# Update all dependencies to latest compatible versions
uv lock --upgrade

# Update a specific package
uv lock --upgrade-package fastapi

Dependency groups

UV supports dependency groups in pyproject.toml for organizing dev, test, docs, etc.

[dependency-groups]
dev = [
    "ruff>=0.9",
    "mypy>=1.14",
]
test = [
    "pytest>=8.3",
    "pytest-cov>=6.0",
    "pytest-asyncio>=0.25",
]
docs = [
    "mkdocs>=1.6",
    "mkdocs-material>=9.5",
]
# Install specific groups
uv sync --group test
uv sync --group dev --group test

# Install all groups
uv sync --all-groups

Running Code with UV

uv run — the main command

uv run ensures your virtual environment exists and is synced, then runs your command inside it. No manual activation needed.

# Run a Python script
uv run python app.py

# Run a module
uv run python -m pytest
uv run python -m mypy src/

# Run an installed CLI tool
uv run fastapi dev
uv run ruff check .

# Run with extra inline dependencies (not in pyproject.toml)
uv run --with rich python -c "from rich import print; print('[bold]Hello![/bold]')"

uvx — run tools without installing

uvx is like npx — it runs Python CLI tools in an isolated environment without polluting your project.

# Run tools without installing them into your project
uvx ruff check .
uvx black --check .
uvx mypy src/
uvx isort --check .

# Run a specific version
uvx ruff@0.9.0 check .

# Run a tool from a package with a different name
uvx --from httpie http GET example.com

Scripts with inline dependencies

UV supports PEP 723 inline script metadata — declare dependencies at the top of a single-file script.

# /// script
# requires-python = ">=3.12"
# dependencies = [
#     "httpx",
#     "rich",
# ]
# ///

import httpx
from rich import print

resp = httpx.get("https://api.github.com/zen")
print(f"[bold green]{resp.text}[/bold green]")
# UV reads the metadata and installs deps automatically
uv run my_script.py

Linting & Formatting — Ruff Docs

Ruff is an extremely fast Python linter and formatter (also from Astral). It replaces flake8, isort, black, pyflakes, and many more — in a single tool.

Setup

# Add to dev dependencies
uv add --group dev ruff

# Or run without installing
uvx ruff check .
uvx ruff format .

Configuration in pyproject.toml

[tool.ruff]
target-version = "py312"
line-length = 88

[tool.ruff.lint]
select = [
    "E",     # pycodestyle errors
    "W",     # pycodestyle warnings
    "F",     # pyflakes
    "I",     # isort
    "N",     # pep8-naming
    "UP",    # pyupgrade
    "B",     # flake8-bugbear
    "SIM",   # flake8-simplify
    "TCH",   # flake8-type-checking
]
ignore = [
    "E501",  # line too long (formatter handles this)
]

[tool.ruff.lint.isort]
known-first-party = ["my_app"]

[tool.ruff.format]
quote-style = "double"
indent-style = "space"

Common commands

# Lint
uv run ruff check .              # Check for issues
uv run ruff check . --fix        # Auto-fix what it can

# Format
uv run ruff format .             # Format all files
uv run ruff format --check .     # Check without changing

# Both in sequence
uv run ruff check . --fix && uv run ruff format .

Testing — pytest Docs

Setup

# Add test dependencies
uv add --group test pytest pytest-cov pytest-asyncio

Configuration in pyproject.toml

[tool.pytest.ini_options]
testpaths = ["tests"]
python_files = "test_*.py"
python_functions = "test_*"
addopts = "-v --tb=short"
asyncio_mode = "auto"          # If using pytest-asyncio

Running tests

# Run all tests
uv run pytest

# Run specific file or test
uv run pytest tests/test_main.py
uv run pytest tests/test_main.py::test_login

# With coverage
uv run pytest --cov=src/my_app --cov-report=term-missing

# Run only tests matching a pattern
uv run pytest -k "auth"

# Stop on first failure
uv run pytest -x

Fixtures (conftest.py)

# tests/conftest.py
import pytest
from my_app.main import create_app


@pytest.fixture
def app():
    """Create application for testing."""
    return create_app(testing=True)


@pytest.fixture
def client(app):
    """Create test client."""
    from httpx import AsyncClient, ASGITransport
    transport = ASGITransport(app=app)
    return AsyncClient(transport=transport, base_url="http://test")


@pytest.fixture
def sample_user():
    """Sample user data for tests."""
    return {
        "email": "test@example.com",
        "name": "Test User",
    }

Example test

# tests/test_main.py
import pytest


async def test_health_check(client):
    resp = await client.get("/health")
    assert resp.status_code == 200
    assert resp.json() == {"status": "ok"}


def test_create_user(sample_user):
    assert sample_user["email"] == "test@example.com"


@pytest.mark.parametrize("input,expected", [
    ("hello", "HELLO"),
    ("world", "WORLD"),
])
def test_uppercase(input, expected):
    assert input.upper() == expected

Type Checking — mypy & Pydantic Docs

mypy setup

uv add --group dev mypy
[tool.mypy]
python_version = "3.12"
strict = true
warn_return_any = true
warn_unused_configs = true
plugins = ["pydantic.mypy"]   # If using Pydantic

[[tool.mypy.overrides]]
module = "tests.*"
disallow_untyped_defs = false
# Run type checking
uv run mypy src/

Modern type hints (Python 3.12+)

# Use built-in generics — no imports needed
def get_users() -> list[dict[str, str]]:
    ...

# Union with | operator
def find(id: int) -> User | None:
    ...

# Type aliases with `type` statement (3.12+)
type UserMap = dict[str, list[User]]

# TypedDict for structured dicts
from typing import TypedDict

class Config(TypedDict):
    debug: bool
    port: int
    host: str

Pydantic for runtime validation

from pydantic import BaseModel, EmailStr, Field


class UserCreate(BaseModel):
    name: str = Field(min_length=1, max_length=100)
    email: EmailStr
    age: int = Field(ge=0, le=150)


class UserResponse(BaseModel):
    id: int
    name: str
    email: str

    model_config = {"from_attributes": True}  # Works with ORM objects

# Python Rules

- Use Python 3.12+ features (type aliases, match statements, f-strings)
- Always use async def for route handlers and database calls
- Use Pydantic BaseModel, not raw dicts, for structured data
- Import order: stdlib, third-party, local (Ruff handles this via isort)
- Tests use pytest fixtures, not setUp/tearDown methods

Common Workflows

Start a new FastAPI project from scratch

# 1. Create project
uv init my-api
cd my-api

# 2. Pin Python
uv python pin 3.12

# 3. Add dependencies
uv add fastapi[standard] sqlalchemy asyncpg pydantic-settings alembic
uv add --group dev pytest pytest-cov pytest-asyncio ruff mypy httpx

# 4. Create the app
mkdir -p src/my_api/routes src/my_api/models tests

# 5. Run the dev server
uv run fastapi dev src/my_api/main.py

Start a new CLI tool

# 1. Create project
uv init my-tool
cd my-tool

# 2. Add dependencies
uv add typer rich
uv add --group dev pytest ruff

# 3. Add entry point to pyproject.toml
# [project.scripts]
# my-tool = "my_tool.main:app"

# 4. Run during development
uv run my-tool --help

Add to an existing project (has requirements.txt)

# 1. Init UV in the existing directory
uv init

# 2. Import existing requirements
uv add $(cat requirements.txt | grep -v "^#" | grep -v "^$")

# 3. Remove old files (optional)
rm requirements.txt setup.py setup.cfg

# 4. Sync and verify
uv sync
uv run python -c "import my_app; print('OK')"

Build & publish a package

# Build the package
uv build

# Publish to PyPI
uv publish

# Or publish to a private registry
uv publish --index-url https://my-registry.com/simple/
